# Web 设计质量标准

## 避免模板感

前端产出应为该产品专门设计，具备明确的设计意图，而非呈现通用模板感。

### 禁止使用的模式

- 间距均匀、缺乏层次的默认卡片网格
- 居中标题 + 渐变背景 + 通用 CTA 的标准 hero 布局
- 直接使用库默认样式作为最终设计交付
- 缺乏深度、层次和动效的平面界面
- 所有组件使用统一的圆角、间距和阴影
- 白底灰字加单一强调色的简单配色方案
- 侧边栏 + 卡片 + 图表堆砌的套路式仪表盘，缺乏设计观点
- 不经选择直接使用默认字体栈

### 必须满足的设计要求

每个值得投入的前端页面，以下十条中至少满足四条：

1. 通过比例对比建立清晰的层次结构
2. 间距具备节奏感，避免均匀分布
3. 通过叠加、阴影、表面层次或动效营造深度
4. 字体经过选择和搭配，具有性格
5. 颜色使用以语义驱动，而非纯装饰
6. hover、focus、active 状态经过专门设计
7. 在合适的场景打破网格，使用编辑式或便当式布局
8. 根据视觉方向，适当加入纹理、颗粒感或氛围
9. 动效用于阐明变化逻辑，而非分散注意力
10. 数据可视化从设计系统层面统一规划

## 前端编码前的准备

1. 确定明确的设计方向。避免使用"简洁极简"等模糊表述。
2. 有意识地构建调色板。
3. 有意识地选择字体。
4. 收集至少几份真实参考案例。
5. 使用 ECC 设计/前端相关技能辅助。

## 可选设计方向

- 编辑式 / 杂志风格
- 新粗野主义
- 具备真实深度的玻璃拟态
- 暗色奢华或浅色奢华，对比度需克制
- 便当式布局
- 滚动叙事
- 3D 集成
- 瑞士 / 国际主义风格
- 复古未来主义

不要无脑默认暗色模式。根据产品特性选择最合适的方向。

## 组件自查清单

- [ ] 是否避免了默认 Tailwind 或 shadcn 模板感？
- [ ] hover/focus/active 状态是否经过设计？
- [ ] 是否使用层次感而非均匀强调？
- [ ] 截图是否具备真实产品的可信度？
- [ ] 如果支持浅色和暗色双主题，两者是否都经过精心设计？
