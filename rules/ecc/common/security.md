# 安全指南

## 每次提交前必查

提交代码前，逐项确认：

- [ ] 无硬编码的密钥（API 密钥、密码、Token）— 最高优先级
- [ ] 所有用户输入已校验
- [ ] SQL 使用参数化查询，非字符串拼接
- [ ] HTML 已清理，防止 XSS
- [ ] CSRF 保护已启用
- [ ] 认证和授权逻辑正确
- [ ] 所有端点已添加速率限制
- [ ] 错误信息不会泄露敏感数据

## 密钥管理

- 严禁将密钥写入源码
- 必须使用环境变量或专用密钥管理工具
- 启动时验证所需密钥是否齐全
- 一旦怀疑密钥泄露，立即轮换

## 安全问题响应

发现安全问题时：
1. 立即停止当前操作
2. 调用 **security-reviewer** Agent 分析
3. CRITICAL 级别问题必须在继续之前修复
4. 轮换所有已泄露的密钥
5. 扫描整个代码库，检查是否存在同类问题
