# 性能优化

## 模型选择策略

**Haiku 4.5**（约为 Sonnet 90% 能力，成本降至 1/3）：
- 高频调用、负载较轻的 Agent
- 结对编程、代码生成
- 多 Agent 系统中的 Worker 角色

**Sonnet 4.6**（当前编码能力最强）：
- 主力开发任务
- 编排多 Agent 工作流
- 复杂编码任务

**Opus 4.5**（推理能力最深）：
- 重大架构决策
- 需要深度推理的场景
- 研究与分析任务

## 上下文窗口管理

以下场景应避免用满上下文窗口的最后 20%：
- 大规模重构
- 跨文件功能实现
- 调试复杂交互

以下场景对上下文敏感度较低：
- 单文件编辑
- 独立工具函数编写
- 文档更新
- 简单 Bug 修复

## 深度思考与 Plan 模式

深度思考默认启用，最多预留 31,999 个 token 用于内部推理。

深度思考控制方式：
- **开关**：Option+T（macOS） / Alt+T（Windows/Linux）
- **配置**：在 `~/.claude/settings.json` 中设置 `alwaysThinkingEnabled`
- **预算上限**：`export MAX_THINKING_TOKENS=10000`
- **查看过程**：按 Ctrl+O 查看思考内容

处理需要深度推理的复杂任务时：
1. 确认深度思考已启用（默认已开启）
2. 开启 **Plan 模式** 以结构化思考
3. 多轮评审确保分析深入
4. 使用分工明确的子 Agent 从不同角度审视

## 构建故障排查

构建失败时：
1. 调用 **build-error-resolver** Agent
2. 分析错误信息
3. 逐步修复，避免一次性改动过多
4. 每次修复后验证结果
