# PHP 编码风格

> 本文件扩展了 [common/coding-style.md](../common/coding-style.md)，补充 PHP 特定的编码约定。

## 标准

- 遵循 **PSR-12** 进行格式化和命名，不应自定义编码标准。
- 应用代码文件顶部应添加 `declare(strict_types=1);`。
- 新代码中所有可行位置均应添加标量类型提示、返回类型和类型化属性。

## 不可变性

- 跨服务边界传递数据时，优先使用不可变 DTO 或值对象。
- 请求/响应载体中应尽可能使用 `readonly` 属性或不可变构造函数。
- 简单映射使用数组即可；涉及业务关键的数据结构应提升为正式的类。

## 格式化

- 使用 **PHP-CS-Fixer** 或 **Laravel Pint** 进行代码格式化。
- 静态分析推荐 **PHPStan** 或 **Psalm**。
- 将 Composer 脚本签入仓库，本地和 CI 执行相同命令，确保环境一致性。

## 导入

- 所有引用的类、接口、trait 均需使用 `use` 语句。
- 避免依赖全局命名空间编写类名，除非项目明确使用完全限定名。

## 错误处理

- 遇到异常情况直接抛出异常；新代码不应再使用 `false`/`null` 进行隐式错误传递。
- 框架/请求的输入在传递给业务逻辑前，应先转换为已验证的 DTO。
