# PHP 模式

> 本文件扩展了 [common/patterns.md](../common/patterns.md)，补充 PHP 特定的设计模式。

## 轻量控制器，显式服务

- 控制器仅处理传输层事务：认证、验证、序列化、状态码。
- 业务规则移至应用/领域服务中，使其无需启动 HTTP 引导即可直接测试。

## DTO 与值对象

- 具有复杂结构的关联数组应替换为 DTO，尤其是请求、命令和外部 API 载体。
- 金额、标识符、日期范围等具有约束概念的类型应使用值对象。

## 依赖注入

- 依赖接口或窄范围的服务契约，而非框架全局对象。
- 通过构造函数传入协作者，使服务无需服务定位器即可独立测试。

## 边界

- 当模型层不仅是持久化时，应将 ORM 模型与业务决策隔离。
- 第三方 SDK 应包装为轻量适配器，使代码库其余部分依赖自有契约而非第三方接口。
